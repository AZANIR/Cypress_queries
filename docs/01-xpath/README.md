# Тема 1: XPath та CSS селектори в Cypress

## Огляд теми

Ця тема вводить два основні способи пошуку елементів у Cypress: **CSS селектори** (нативна підтримка) та **XPath селектори** (через плагін cypress-xpath). Ви дізнаєтесь про переваги та недоліки кожного підходу та коли використовувати той чи інший.

## Що ви вивчите

- Основні типи CSS селекторів (класи, ID, атрибути, теги)
- Комбіновані CSS селектори та псевдокласи
- Встановлення та налаштування cypress-xpath
- Базові XPath селектори та їх синтаксис
- Порівняння CSS та XPath підходів
- Коли використовувати CSS, а коли XPath

## CSS селектори - основа Cypress

Cypress нативно підтримує CSS селектори, які є основним інструментом для пошуку елементів. CSS селектори швидкі, ефективні та добре підтримуються браузерами.

### Основні типи CSS селекторів:

1. **Класи** - `.class-name` (найпоширеніший спосіб)
2. **ID** - `#element-id` (уникайте, якщо можливо)
3. **Теги** - `tag-name` (найменш специфічний)
4. **Атрибути** - `[attribute="value"]` (стабільний спосіб)

### Переваги CSS селекторів:

- ✅ Нативна підтримка в Cypress
- ✅ Швидкі та ефективні
- ✅ Добре оптимізовані браузерами
- ✅ Прості та читабельні
- ✅ Підтримують псевдокласи

### Недоліки CSS селекторів:

- ❌ Обмежені можливості для пошуку за текстом
- ❌ Складніші для навігації по DOM дереву
- ❌ Менш гнучкі для складних умов

## XPath селектори - альтернативний підхід

XPath (XML Path Language) - це мова для навігації по XML/HTML документах. У Cypress XPath підтримується через плагін `cypress-xpath`.

### Встановлення cypress-xpath:

```bash
npm install -D cypress-xpath
```

### Налаштування:

Додайте в `cypress/support/e2e.js`:
```javascript
import 'cypress-xpath'
```

### Переваги XPath:

- ✅ Потужні можливості для пошуку за текстом
- ✅ Гнучкі умови та фільтри
- ✅ Зручна навігація по DOM дереву
- ✅ Підтримка складних виразів

### Недоліки XPath:

- ❌ Потребує додаткового плагіна
- ❌ Менш ефективний за CSS
- ❌ Плагін cypress-xpath є deprecated
- ❌ Складніший синтаксис

## Коли використовувати CSS, а коли XPath?

### Використовуйте CSS селектори для:

- Пошуку за класами, ID, атрибутами (90% випадків)
- Простих структурних пошуків
- Максимальної продуктивності
- Стандартних випадків

### Використовуйте XPath селектори для:

- Пошуку за складним текстовим вмістом
- Складних умов з множинними критеріями
- Навігації по складних DOM структурах
- Випадків, коли CSS не справляється (10% випадків)
